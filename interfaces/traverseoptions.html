Optional bfs

bfs: undefined | false | true
  • Defined in index.ts:37

true if you want to traverse in a breadth-first manner. This will cause the mutation function to be called first with the root schema, moving down the subschemas until the terminal subschemas.

Optional mergeNotMutate

mergeNotMutate: undefined | false | true
  • Defined in index.ts:25

Set this to true if you want to merge the returned value of the mutation function into the original schema.

Optional mutable

mutable: undefined | false | true

true if you want the original schema that was provided to be directly modified by the provided mutation/merge function To preserve cyclical refs this is necessary.
